**Overview**

Genome-wide association studies (GWAS) are a genetic approach to finding
association of specific genetic variations with diseases by comparing
thousands of variants across many genomes. GWAS data is essential for
aiding researchers in further understanding the evolution of organisms
through identifying common single-nucleotide polymorphisms (SNPs) or
point mutations in a genome(1). SNPs are the most common mutation and
utilized in GWAS as genetic markers of highly variable regions in an
organism's genome that are often linked to disease states and drug
interactions. When a gene is associated with a disease, SNPs are often found in the surrounding region, which researchers can use as a genetic marker for the disease (2). Researchers conducting GWAS use these regions with SNPs as
genetic markers for identifying genes subsequently coding regions that contain alleles associated with the disease state. Most GWAS studies are conducted with
humans, combining diseased and healthy individuals so researchers can
identify variable regions that are likely contributing to the disease in
individuals to understand modality and paths to treatment of the
disease. GWAS can be completed for any species and are often used for
deadly organisms such as *Plasmodium falciparum*, a parasite that causes
malaria.

One way to visualize GWAS results is through regional Manhattan plots
generated by the web-based tool LocusZoom. LocusZoom uses GWAS data in
tandem with linkage disequilibrium metrics to visualize the intensity of
a SNP association call to understand what genes are in the region,
whether the SNP is coding or non-coding, and how many linked SNPs are in
the region(3). Linkage disequilibrium (LD) describes how frequently two
alleles occur together, suggesting that it is more likely that the SNP
region between them is conserved and frequently occurs(4). LocusZoom
Manhattan plots display p-values on a logarithmic scale along a
user-specified chromosomal region. The higher the association of a SNP,
the larger the magnitude of association. Currently, LocusZoom is built
only for Human GWAS data, and there isn't a tool that visualizes GWAS LD
data through Manhattan plots for *Plasmodium falciparum*.

LocusZoom only supports human reference genomes GRCh37 and GRCh38, which
have 22 somatic chromosomes and 2 sex chromosomes, X and Y.
Additionally, all coding regions have been named and identified in the
human genome. However, *P. falciparum* (3D7) only has 14 chromosomes,
and not all coding regions have been named and their functionality
identified. Here, we present a LocusZoom based visualization tool that
generates LD Manhattan plots of *P. falciparum* SNPs from GWAS using
updated chromosomes, coding regions, and LD metrics with the aim of
allowing users to identify SNPs associated with malaria.

**Context**

By analyzing the data from a GWAS, we
can determine linkage disequilibrium, and SNPs that occur
more frequently in Plasmodium falciparum that can be used to battle this
devastating parasite. There are plenty of tools
to generate LocusZoom plots to study human GWAS
and linkage disequilibrium, however, there are none available for
Plasmodium falciparum. For our project, we are going to create a tool
that directly creates LocusZoom plots for Plasmodium falciparum, from
GWAS data and a sample genotype. Once this tool is developed, scientists
can use this tool to identify SNPs and linkage disequilibrium, to
help identify solutions to fight against this parasite.

sample and GWAS data will come from Dr. Wheeler. We will generate PLINK
files that we can use for LD calculations from the GWAS data. After
this, we will have the necessary files to begin converting the LocusZoom
code to fit *P. falciparum*. To do this, we will need to identify the
hard-coding regions where they implement the human genome, and change
the linear regression to logistic regression. After, we will need to
change these hard-coding regions to fit *P. falciparum*. In addition to
this, we will fix the LD calculations done by file in the repository so
that it matches the *P. falciparum* genome and the GWAS + sample data.
Once the LocusZoom plotting has been fixed for *P. falciparum*, we will
troubleshoot and run tests with our sample data to make sure that we
have executed it properly. User's should be able to use the GWAS data
and provided unique gene in accordance with their own sample data and
specified SNP to execute the workflow. Users will use the provided files
in our repository to generate the LD and logistic regression association
file, and then run the LocusZoom plotting code to generate the desired
Manhattan plot.
